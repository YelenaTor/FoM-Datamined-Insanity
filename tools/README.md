# Discovery Tools

Python scripts that extract FoM internals from a live game install and regenerate the docs.

## Requirements

- Python 3.9+
- Fields of Mistria installed
- `gml_discovery.txt` — a YYTK script dump of `FieldsOfMistria.exe` (required for script-based tools)
- `__fiddle__.json` — present in the FoM runtime folder (auto-generated by the game on first run)

## `--game-path` argument

All tools accept `--game-path` pointing at your FoM game folder. The tools search for:
- `__fiddle__.json` at `<game-path>/runtime/Fields of Mistria/__fiddle__.json` or `<game-path>/__fiddle__.json`
- `gml_discovery.txt` at `<game-path>/.temp/gml_discovery.txt` or `<game-path>/gml_discovery.txt`

## Tools

| Script | What it produces | Source needed |
|--------|-----------------|---------------|
| `query_fiddle.py` | Interactive query tool | `__fiddle__.json` |
| `discover_resources.py` | Resource node type IDs, tool scripts | `gml_discovery.txt` |
| `discover_areas.py` | Location IDs, room transition scripts | both |
| `discover_npcs.py` | NPC data, portrait emotions, NPC scripts | both |
| `discover_items.py` | Item IDs from gift/store/forageable data | `__fiddle__.json` |
| `discover_crafting.py` | Crafting station scripts, recipe data | both |
| `discover_scripts.py` | Full 2340-script GML catalog | `gml_discovery.txt` |

## Output

Tools write to `../raw/<tool-name>.txt`. The `raw/` folder is gitignored. Once you verify the output is correct for the current FoM version, update the relevant `docs/` Markdown file and open a PR.

## Generating `gml_discovery.txt`

This requires a YYTK script that dumps all GML script entries from the game binary. See the FoM modding community Discord for instructions on running a YYTK headless dump.
